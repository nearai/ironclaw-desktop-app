# R53 — Mermaid + Plotly + KaTeX renderers in MarkdownView

**Lane**: A3 (codex)
**Branch**: `codex/r53-multimodal-renderers`
**Depends on**: nothing

## Context

LLM responses routinely include Mermaid diagrams, math, and (rarely
but increasingly) Plotly JSON specs. Right now they render as
code blocks. Users have to copy-paste into external viewers.

This task wires three renderers into `MarkdownView.svelte`:

1. **Mermaid** — `mermaid` package. Detect ```` ```mermaid ```` blocks.
2. **KaTeX** — `katex` package. Detect `$inline$` and `$$display$$`
   math.
3. **Plotly** — `plotly.js-dist-min` package. Detect
   ```` ```plotly ```` blocks (whose contents are a JSON Plotly spec).

All three lazy-load on first use — adding them to the static bundle
would push us over the size budget. The dynamic-import pattern below
keeps the entry chunk untouched.

## Owned files (exclusive write access)

- `src/lib/components/MarkdownView.svelte` — modify the rendering

## Approach

Three sub-renderers, each a separate Svelte component:

### `Mermaid.svelte`

```svelte
<script lang="ts">
  interface Props { source: string }
  let { source }: Props = $props();
  let svg = $state<string | null>(null);
  let error = $state<string | null>(null);

  $effect(() => {
    let cancelled = false;
    (async () => {
      try {
        const mermaid = (await import('mermaid')).default;
        mermaid.initialize({ startOnLoad: false, theme: 'dark' });
        const id = `mmd-${Math.random().toString(36).slice(2, 10)}`;
        const { svg: out } = await mermaid.render(id, source);
        if (!cancelled) svg = out;
      } catch (err) {
        if (!cancelled) error = (err as Error).message;
      }
    })();
    return () => { cancelled = true };
  });
</script>

{#if error}
  <pre class="text-xs text-red-300 border border-red-500/30 rounded p-2">
    Mermaid render failed: {error}
  </pre>
{:else if svg}
  {@html svg}
{:else}
  <div class="text-xs text-text-muted italic">Rendering diagram…</div>
{/if}
```

### `Math.svelte`

```svelte
<script lang="ts">
  interface Props { source: string; display?: boolean }
  let { source, display = false }: Props = $props();
  let html = $state<string | null>(null);
  let error = $state<string | null>(null);

  $effect(() => {
    let cancelled = false;
    (async () => {
      try {
        const katex = (await import('katex')).default;
        await import('katex/dist/katex.min.css');
        const out = katex.renderToString(source, {
          displayMode: display,
          throwOnError: false,
          output: 'html'
        });
        if (!cancelled) html = out;
      } catch (err) {
        if (!cancelled) error = (err as Error).message;
      }
    })();
    return () => { cancelled = true };
  });
</script>

{#if error}
  <code class="text-xs text-red-300">[math: {error}]</code>
{:else if html}
  <span class={display ? 'block my-2' : 'inline'}>{@html html}</span>
{:else}
  <code class="text-xs text-text-muted">{source}</code>
{/if}
```

### `Plotly.svelte`

```svelte
<script lang="ts">
  interface Props { source: string }
  let { source }: Props = $props();
  let host = $state<HTMLDivElement | null>(null);
  let error = $state<string | null>(null);

  $effect(() => {
    if (!host) return;
    let cancelled = false;
    (async () => {
      try {
        const spec = JSON.parse(source);
        const Plotly = (await import('plotly.js-dist-min')).default;
        if (cancelled || !host) return;
        await Plotly.newPlot(host, spec.data, spec.layout ?? {}, {
          responsive: true,
          displayModeBar: false
        });
      } catch (err) {
        if (!cancelled) error = (err as Error).message;
      }
    })();
    return () => { cancelled = true };
  });
</script>

{#if error}
  <pre class="text-xs text-red-300 border border-red-500/30 rounded p-2">
    Plotly render failed: {error}
  </pre>
{/if}
<div bind:this={host} class="w-full"></div>
```

### `MarkdownView.svelte` integration

Find the existing fenced-block rendering path. Add detection for
`info === 'mermaid' | 'plotly'` BEFORE the generic code-block highlight
path. For math, detect via a custom markdown-it plugin OR a simple
regex pass post-render — simpler approach is regex post-render on
`text/plain` segments.

Constraints:
- KaTeX CSS is required for math to render — lazy-import it inside
  `Math.svelte` (CSS imports inside `import()` work in Vite).
- Mermaid v10+ supports dark theme natively — pass it once in init.
- Plotly is HUGE (~2.5 MB unminified). The `dist-min` variant is
  ~1.2 MB. Lazy-loading keeps it OUT of the entry chunk — verify with
  `npm run build && du -sh build/_app/immutable/chunks/plotly*`.

## Acceptance

1. `npm run check` → 0 errors.
2. `npm run test` → all green. New test cases:
   - Markdown with a `mermaid` code block renders a `<svg>` after
     async load.
   - Markdown with `$x^2$` renders a `.katex` element.
   - Markdown with a `plotly` code block mounts a div with a
     `plotly` class on the host.
   - Each renderer falls back to a code-block on parse error.
3. `npm run build` → succeeds.
4. `bash scripts/check-bundle-size.sh` → entry chunk does NOT include
   mermaid/katex/plotly bytes. Use `find build/_app/immutable -name
   '*mermaid*' -o -name '*katex*' -o -name '*plotly*'` to confirm
   they exist as separate chunks.
5. Manual: paste a markdown payload like

   ````
   ```mermaid
   graph TD; A-->B
   ```

   The mass is $E=mc^2$ in display:

   $$\int_0^\infty e^{-x^2} dx = \frac{\sqrt{\pi}}{2}$$

   ```plotly
   {"data":[{"x":[1,2,3],"y":[2,4,6],"type":"scatter"}],"layout":{"title":"Demo"}}
   ```
   ````

   into a chat composer, send, verify all three render in the response.

## Out of scope

- LaTeX block elements beyond inline + display math (matrices,
  align environments) — KaTeX supports them; just don't add custom
  CSS for them.
- Mermaid live-edit (the renderer is read-only).
- Plotly interactivity (`displayModeBar: false` for now).
- Diagrams-as-data (Vega-Lite) — separate lane.

## Notes

- Mermaid sometimes flickers on theme change because it re-renders
  the SVG. That's acceptable; out-of-scope.
- KaTeX failure shows the raw source as a `<code>` — fine.
- The bundle-size script will need updating if Plotly's chunk lands
  near the budget edge. That's still in A1's lane though — don't
  preemptively edit it here.
